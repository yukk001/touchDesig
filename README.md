# touchDesign
设计模式概念练习demo\
设计模式的主要目的就是避免更多无所谓的修改；\
在功能稳定的情况下，变更需求的时候，是以新增的形式实现，避免修改。\
设计模式主要针对的就是软件变更\
面向对象编程，面向过程编程，面向函数编程，指令式编程。\
面向对象的设计原则：

### 单一职责原则：
就一个类而言，应该仅有一个引起它变化的原因
### 开放封闭原则：
软件实体应该是（类，函数，模块）可以扩展的，但是不可修改
### 依赖倒置原则：
抽象不应该依赖细节，细节应该依赖于抽象
### 接口隔离原则：
多个专用接口优于一个单一的通用接口
### 里氏替换原则；
子类必须能够替换掉它的父类
